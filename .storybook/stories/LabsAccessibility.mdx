# Labs: Accessibility — Key screens and checks

import { Meta } from '@storybook/blocks';

<Meta title="Labs/Accessibility" />

What this lab covers
- Basic a11y checks on key screens and components

Stories/Docs to use
- .storybook/test-runner.ts — Axe checks (targeted)
- Use Storybook Test Runner to run a11y on Labs pages and key screens

Exercises
1) Labels and roles
   - Ensure interactive elements have accessible names and roles
2) Focus and navigation
   - Confirm focus order and visibility; ensure keyboard nav works in web
3) Contrast and touch targets
   - Validate text contrast meets WCAG; confirm comfortable touch targets
